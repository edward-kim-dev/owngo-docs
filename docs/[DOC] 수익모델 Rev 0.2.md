# OwnGo 핵심 수익 모델 기획안 (AI 에이전트 기반)

**핵심 전제:** 자연어를 통한 즉각적인 시스템 구축은 사용자에게 강력한 편의성을 제공함과 동시에, 축적되는 데이터와 자동화된 워크플로우로 인해 **강력한 락인(Lock-in) 효과**를 창출합니다. 이를 기반으로 '생성 ➡️ 실행 ➡️ 확장'의 단계별 수익 구조를 설계합니다.

---

## 1. 구축의 복잡도에 따른 구독 모델 (SaaS Tiers)
AI 에이전트가 생성하는 '시스템과 데이터베이스의 깊이'를 기준으로 플랜을 차등화합니다.

* **Starter (Free/Low-cost): 단순 데이터베이스 기반**
  * 방문자 기록, 연차 신청서, 비품 요청 등 1~2개의 단일 테이블로 구성되는 단순 워크플로우 생성 허용.
  * 플랫폼 진입장벽을 낮추고 바이럴을 유도하는 모객용 티어.
  * owngo 자체 서버(Free Tier) or 국내 클라우드에서 운영(지원받을경우)
* **Business (Core Revenue): 관계형 DB 및 ERP 모듈 연동**
  * 인사 데이터와 지출 결의, 재고 관리와 영업 수주 등 데이터 간의 관계(Relation)가 필요한 비즈니스 수준의 구조 설계 지원. 
  * 국세청 휴폐업 조회, 은행 계좌 연동 등 외부 API 플러그인 연결 권한 부여.
* **Enterprise: 전용 온톨로지 및 에이전트 튜닝**
  * 기업 내부의 방대한 규정이나 특수한 비즈니스 로직을 온톨로지로 별도 구축.
  * 해당 기업의 데이터에만 최적화된 전용 AI 에이전트 및 폐쇄형 환경 제공.
  * 기업의 요구사항에 맞춰 운영.(온프레미스, 클라우드 등 비용 별도)

## 2. RPA 실행에 대한 컴퓨팅 리소스 과금 (Usage-Based)
시스템 '생성'의 허들은 낮추되, 시스템이 '동작'할 때마다 발생하는 서버 리소스(컴퓨팅)에 대해 과금하는 종량제 모델입니다.

* **운영 방식:** RPA 커스텀 지원 전면 차단 + 플랫폼 기반 스케줄링. AI가 구축한 RPA(자동화 워크플로우)가 실행될 때마다 '액션 크레딧' 차감.
  * *예: "재고 소진 시 거래처 발주 메일 전송" (1 크레딧)*
  * *예: "매일 아침 9시 전일 매출 요약 슬랙 알림" (2 크레딧)*
* **기대 효과:** 고객의 비즈니스가 성장하여 워크플로우 활용 빈도가 높아질수록 플랫폼의 수익도 비례하여 증가하는 동반 성장형 모델.

## 3. 인프라 배포 환경에 따른 라이선스 수익 (Hosting vs. Self-Hosted)
ERP/RPA 시스템의 특성상 보안과 데이터 통제권을 중시하는 기업을 타겟으로 한 인프라 분리 과금 전략입니다.

* **Managed Cloud (구독형):** OwnGo의 클라우드 환경에서 모든 서비스가 호스팅되는 기본 형태 (월/연간 구독).
* **Self-Hosted (고액 라이선스 + SLA):** 보안에 민감하거나 자체 Linux/Windows 서버 환경을 보유한 기업 대상. AI 에이전트가 설계한 아키텍처와 DB를 독립적인 **Docker 컨테이너 이미지**로 추출(Export)하여 고객사 인프라에 직접 배포. 초기 도입비(라이선스)와 연간 유지보수(SLA) 계약을 통한 고수익 창출.

## 4. 'OwnGo Builder' API 및 화이트라벨링 (B2B2B)
"자연어 해독 ➡️ 오픈소스 ERP 매핑"이라는 OwnGo의 코어 엔진 자체를 타 비즈니스 솔루션에 공급하는 생태계 확장 모델입니다.

* **운영 방식:** 타 SaaS 기업, 협회 전용 프로그램, 레거시 시스템 등에 OwnGo의 대화형 에이전트 UI/API만 모듈 형태로 탑재.
* **기대 효과:** OwnGo 브랜드를 전면에 내세우지 않더라도(화이트라벨링), 타사의 고객 트래픽을 활용하여 API 호출량 기반의 안정적인 인프라 수익 확보 가능.

## 5. 'OwnGo Marketplace' 
스스로 성장하는 플랫폼 

* **운영 방식:** OwnGo에 solver로 등록된 개발자 및 기업이 개발한 프로그램을 OwnGo 마켓플레이스에 등록하고 판매. 사용자는OwnGo를 통해 다양한 프로그램을 구매하고 응용하여 자신만의 시스템을 구축할 수 있음.
* **기대 효과:** OwnGo는 마켓플레이스에 등록된 솔루션의 판매 수수료를 통해 수익을 창출할 수 있음. 또한 다양한 프로그램을 통해 OwnGo 생태계를 확장하고 사용자의 참여를 유도할 수 있음.

### 💡 전략적 핵심 (The "Gotcha") : Lock-in과 Export의 딜레마
* **진입은 쉽게:** 초기에는 사용자가 자연어로 시스템을 제약 없이 만들어보도록 허용하여 락인(Lock-in) 효과를 극대화합니다.
* **이탈은 비용으로:** 축적된 데이터(엑셀 형태)의 반출은 허용하되, **AI가 구성한 '워크플로우 로직'과 '아키텍처' 자체를 내보내기(Export)하거나 외부 시스템과 연동(Webhook/API)하려는 시점**에 강력한 과금 게이트(Business 이상 플랜)를 설정하는 것이 B2B 수익화의 핵심입니다.
